功能列表：
  1. 能够保存多个 指定用户 / 收藏夹(公共收藏夹&私人收藏夹) / 话题精华 /专栏/ 问题 /指定回答 的答案并输出为Epub电子书
  2. 电子书中答案按照问题进行整合，问题按其下答案的总赞同数排序，专栏按发布日期倒序排列
  3. 当用户将答案标注为『禁止转载』时助手将不会读取该答案
  4. 跨平台，Windows用户直接双击运行zhihuHelp.py即可，Mac/Linux用户请参考说明中的教程（非常简单）
  5. 增量更新。在每一次运行时程序会录入所有新增和修改过的答案，但不会把答案在数据库里删除，所以，利用这个程序可以实现对知乎答案的永久保存
  6. 自动检测更新
  7. 感谢 @林西豪 、 @Molly麥 、 @刘三清、@官姚、@骆启明、@夏小山、 @公子妖 、 @Suji Yan 、百度网盘网友 千尘千叶 、 知友@宁晓卿 、 @李婷婷 、 @黄淇 、 @郑智彬、 @肖智博 、@冯舒勤 、@张佳硕 、 @梨花雨凉 、 @Popeye 、 @圈圈 、 @其他情绪 、 @陈思吉 、 @Philip Matthew 、 @鱼藻君 的捐赠与 @李奥 的推荐，谢谢大家的认可！谢谢(∩_∩)

使用方法：

Windows用户：
收集 指定用户 / 收藏夹(公共收藏夹&私人收藏夹) / 话题精华 的答案：
  1. 安装python2.7：Windows32位用户请下载python2.7.9_32位_Windows 然后安装一路下一步/Next即可，Windows64位用户请下载 python2.7.9_64位_Windows,也是一路Next/下一步，Mac/Linux用户不用下~系统里面自带了
  2. 下载 知乎助手，解压到任意文件夹
  3. 把待读取的 用户知乎主页地址、收藏夹首页地址、话题地址、问题地址或者答案地址、专栏地址亦或者是专栏文章地址复制粘贴到文件夹中的ReadList.txt里并保存(可参考下面为Mac/Linux用户写的图文攻略)
  4. 双击运行zhihuHelp.py，开始时会让输入最大线程数，建议输入10或更小的数（光缆用户请勇敢的敲进20）
  5. 运行完毕后生成的电子书与网页均在知乎答案集锦文件夹中，Epub格式的电子书可在刷完多看系统的Kindle上、手机多看客户端以及Chrome的Readium 扩展上阅读
  6. WinXP~Win8均能直接运行，不能运行请留言反馈。


Tips：
  1. 生成的电子书默认为带有图片，上网按流量计费的同学或者需要下载大收藏夹的同学可考虑使用无图版进行下载
  2. 针对丢失图片的问题，可以通过设置线程数解决，建议线程数=使用迅雷下载1G以上文件时的稳定网速(kb/s)÷100x5，即100kb/s的网速环境设置成5，200kb/s设置成10，500kb/s设置成25，以此类推
  3. 鉴于源代码已经超过了1500行，所以就不放在这儿了~，
    1. GitHub地址:YaoZeyuan/ZhihuHelp__Python · GitHub
  4. Kindle用户可以通过刷多看系统解决不能读取Epub文件的问题，非常简单
    1. 在官网上下载多看系统并根据要求放入Kindle中
    2. 打开Kindle点按菜单
    3. 重启，完成
  5. 非常不推荐使用mobi转换工具将Epub转换成mobi
    1. 转换过程非常漫长而且没有进度指示（在使用亚马逊官方转换工具的情况下）
    2. 丢失全部排版，当答案比较短时页面会变得掺不忍睹
    3. 刷多看又不麻烦为什么不用多看啊摔
    4. 好吧我是完美主义者

Mac/Linux用户（演示系统为Ubuntu 14.04LTS）：
  1. 首先，将知乎助手下载至桌面上，解压缩，得到『知乎助手1.7.0.2』文件夹

  2. 然后，打开文件夹，将想要收集的链接存入ReadList.txt中，每一行为一本电子书，在同一行内的链接使用$分开，回车换行，可以在每一行链接的末尾可以用#+注释内容的方式添加注释
  3. 打开终端，输入对应命令，切换到知乎助手的文件夹下
    cd Desktop
或者 cd 桌面 (根据操作系统中桌面文件夹的命名方式而定)
-----------------------
cd 知乎助手1.7.0.2   （切换到知乎助手的文件夹中）

  4. 输入
    python zhihuHelp.py
    运行知乎助手
  5. 敲回车，等待运行完毕，over
  6. 运行完成

  7. 若系统里没有自带python2.7的Mac/Linux用户请自行百度如何安装python2.7(基本没这种可能啊摔。。。)
